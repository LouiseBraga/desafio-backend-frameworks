# Desafio Multi-Framework - Back-end Frameworks

Este repositório contém a estrutura básica de dois projetos back-end utilizando diferentes tecnologias, demonstrando a aplicação da arquitetura em camadas (MVC).

## Tecnologias
* **Node.js com Express:** Ambiente leve e flexível.
* **Java com Spring Boot:** Framework robusto e fortemente tipado.

## Arquitetura em Camadas
Ambos os projetos foram organizados seguindo o padrão **MVC (Model-View-Controller)**:
* **Model:** Define a estrutura dos dados (Entidade Usuário).
* **Controller:** Gerencia as requisições HTTP e as respostas.
* **Service (Java) / Routes (Node):** Camadas de lógica e roteamento que conectam o fluxo da aplicação.

## Comparação Técnica

| Critério | Node.js (Express) | Java (Spring Boot) |
| :--- | :--- | :--- |
| **Configuração Inicial** | Muito rápida e manual. | Mais complexa (exige estrutura de pacotes). |
| **Verbosidade** | Código conciso e direto. | Código mais extenso com uso de anotações (@). |
| **Dependências** | Gerenciadas pelo npm (package.json). | Gerenciadas pelo Maven (pom.xml). |

## Conclusão
O Node.js se destaca pela agilidade no desenvolvimento, enquanto o Spring Boot oferece uma estrutura mais segura e organizada para grandes aplicações corporativas.
